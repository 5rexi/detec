# 安全帽与反光衣识别策略建议

## 1. 方案可行性
先用 YOLO 做人检测，再从行人框中裁剪头部和上身给 ResNet 分类是可行的，且工程上容易落地。当前代码已经采用了该两阶段思路（跟踪 + 裁剪 + 分类 + 时序投票）。

## 2. 你的业务目标对应的核心设计
业务目标：
- **绝不能把“已佩戴/已穿着”误报为“未佩戴/未穿着”**（尽量零误报）
- **对“未佩戴/未穿着”只需视频中至少一帧被识别到**（追求召回）

这本质上是一个**非对称代价**问题，应该使用“保守判违规、激进判合规”之外的策略：
- 对“违规告警”设置更高置信度阈值和更严格时序确认，降低误报；
- 允许“未穿戴”单帧候选进入轨迹级累积分数，达到阈值再触发，提升召回。

## 3. 建议的训练策略

### 3.1 多任务与分支建模
- 建议拆分为两个分类头（或两套模型）：
  - 头部：`helmet / no_helmet / uncertain`
  - 上身：`vest / no_vest / uncertain`
- 使用 `uncertain`（不确定）类别吸收遮挡、模糊、过曝、背身等难例，避免模型把难例硬判成违规。

### 3.2 非对称损失（重点）
- 对“把已穿戴判成未穿戴”的错误施加更高惩罚权重：
  - 可用加权交叉熵（class weight）或 focal loss + class weight。
- 若做二分类 logit，可增加保守阈值训练（后处理阈值更关键）：
  - `P(no_helmet) > T_violation` 才记为违规候选，`T_violation` 取高值（如 0.9 起试）。

### 3.3 难例挖掘
- 重点收集“容易误报违规”的样本：
  - 白色安全帽、反光条不明显、强背光、雨天、夜间、遮挡、远距离小目标。
- 训练时提高这些样本采样概率，降低“误报违规”。

### 3.4 轨迹级时序融合（比单帧更关键）
- 你已有 `track + deque` 机制，建议改为“累计证据分数”而非全等判断：
  - 每帧得到 `s_nohelmet = P(no_helmet)`；
  - 轨迹分数 `S_t = alpha * S_{t-1} + s_nohelmet`；
  - 当 `S_t > H` 触发违规；
  - 当 `P(helmet)` 持续很高时快速衰减 `S_t`。
- 这样可以满足“未佩戴只要有一帧命中就能累计触发”，同时减少偶发噪声误报。

### 3.5 阈值分级与告警状态机
- 建议三态：`SAFE / SUSPECT / VIOLATION`
  - `SAFE`：高置信穿戴；
  - `SUSPECT`：单帧或低累计证据；
  - `VIOLATION`：高累计证据触发。
- 对外告警仅在 `VIOLATION` 触发，满足误报控制要求。

## 4. 数据集与标注建议
- 每个行人框同时标注：头部可见性、上身可见性、遮挡率、是否可判定。
- 对不可判定样本标为 `uncertain`，而非强行给 `no_helmet/no_vest`。
- 训练/验证按场景分层（白天/夜间、室内/室外、近景/远景）。

## 5. 评估指标要与业务一致
不要只看整体 accuracy，重点看：
- `FPR_compliance_to_violation`：把合规误报成违规的比例（核心指标，要求极低）；
- `Track-level Recall(no_helmet/no_vest)`：轨迹级召回（允许单帧命中后累积触发）；
- 告警延迟（从出现违规到触发告警的帧数）。

## 6. 与现有实现对齐的改造优先级
1. 保留 YOLO 人检 + 头部/上身裁剪架构。
2. 在分类模型中加入 `uncertain` 类并清洗标注。
3. 把当前固定窗口“全为同类才判定”的逻辑改为累计分数状态机。
4. 使用验证集专门调 `T_violation`、`alpha`、`H`，以“合规误报率最低”为首目标。

## 7. 结论
你的路线是可行的。关键不在是否使用 ResNet，而在于：
- 使用非对称训练目标；
- 引入 `uncertain` 降低错误硬判；
- 用轨迹级证据累积与高阈值告警，严格控制“把合规报成违规”。
